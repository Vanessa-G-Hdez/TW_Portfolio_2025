Payment API Integration Guide

Technical Documentation Project by Vanessa G. Hdez

Overview

The Payment API enables secure online transactions through RESTful endpoints. This document describes authentication, request and response formats, and common error handling.

1. Authentication

To access any endpoint, first obtain an access token.

Endpoint

POST /auth/token

Request Example

{
  "client_id": "your_client_id",
  "client_secret": "your_client_secret"
}

Response Example

{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9",
  "token_type": "Bearer",
  "expires_in": 3600
}

Include the token in the header of subsequent requests:

Authorization: Bearer <access_token>

2. Create a Payment

Endpoint

POST /payments

Request Body

{
  "amount": 250.75,
  "currency": "USD",
  "payment_method": {
    "type": "card",
    "card": {
      "number": "4111111111111111",
      "exp_month": "12",
      "exp_year": "2026",
      "cvv": "123"
    }
  },
  "description": "Order #2458"
}

Response (201 Created)

{
  "payment_id": "pay_01hxyzab123",
  "status": "pending",
  "created_at": "2025-10-15T12:30:00Z"
}
Warning: This is a test environment. Do not use real credit card numbers.

3. Retrieve Payment Status

Endpoint

GET /payments/{payment_id}

Response

{
  "payment_id": "pay_01hxyzab123",
  "status": "completed",
  "amount": 250.75,
  "currency": "USD",
  "description": "Order #2458",
  "completed_at": "2025-10-15T12:32:00Z"
}

4. Error Handling

HTTP Code Description Example Message
400Bad RequestInvalid card number
401UnauthorizedMissing or invalid token
404Not FoundPayment not found
500Internal Server ErrorUnexpected processing error

Sample Error Response

{
  "error": {
    "code": "invalid_request",
    "message": "Card number format is invalid"
  }
}

5. Best Practices

  • Validate input fields on the client side before sending.
  • Use HTTPS for all communications.
  • Implement retry logic for transient 5xx errors.
  • Log both successful and failed requests.
  • Store API credentials securely in environment variables.
